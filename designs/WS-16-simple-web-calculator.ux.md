# 交互设计说明书: 简易计算器

## 1. 业务目标与用户画像
* **目标**：提供一个美观、易用且具备完整基础计算功能的Web端计算器。支持自适应布局，使用户在手机、平板和PC端均能获得优秀的交互体验。
* **核心用户**：需要在各端快速进行基础数学计算的普通用户。

## 2. 页面跳转流程 (User Flow)
* **主流程 (Golden Path)**：
  - 用户打开页面 -> 展现默认状态 (输入区/显示区为 "0")
  - 用户点击/键盘输入数字或操作符 -> 显示区实时反映输入，并按运算逻辑更新计算过程
  - 用户点击或输入 "=" -> 显示区呈现计算结果，并将该笔计算记录计入历史面板
  - 用户点击 "C" 或 "CE" -> 清空/部分清除当前输入，计算器返回至默认状态
* **自适应状态与面板切换流程**：
  - **PC/平板**：历史记录面板常驻或通过侧边栏收折，计算器与历史记录并行呈现。
  - **手机**：历史记录作为浮层/弹窗展示，点击顶部“历史图标”唤起，点击遮罩或右上角“关闭”返回计算器主界面。

## 3. 信息架构与页面布局 (Wireframe)

### 3.1 手机端布局 (Mobile: < 768px)
自适应单栏布局，操作按钮高度适配手指触控区域 (Min 48px)。

```
+--------------------------------------------------+
|  [主题切换: ☀/🌙]                    [历史记录图标] |
+--------------------------------------------------+
|  公式显示区: 125 x 4                             |
|  主数值显示区: 500                                |
+--------------------------------------------------+
|  [ C ]    [ CE ]   [ 退格 ]    [  /  ]           |
|  [ 7 ]    [  8 ]   [  9   ]    [  *  ]           |
|  [ 4 ]    [  5 ]   [  6   ]    [  -  ]           |
|  [ 1 ]    [  2 ]   [  3   ]    [  +  ]           |
|  [ +/-]   [  0 ]   [  .   ]    [  =  ]           |
+--------------------------------------------------+
```

### 3.2 平板与PC端布局 (Tablet & PC: >= 768px)
宽屏两栏布局，计算区与历史记录面板并行，提升大屏空间利用率。

```
+-----------------------------------------------------------------+
|  [主题切换: ☀/🌙]                                                |
+------------------------------------+----------------------------+
|  公式显示区: 125 x 4               | 历史记录面板                |
|  主数值显示区: 500                  |                            |
+------------------------------------+ 125 x 4 = 500              |
|  [ C ]    [ CE ]   [ 退格 ] [  /  ]| 45 + 55 = 100              |
|  [ 7 ]    [  8 ]   [  9   ] [  *  ]|                            |
|  [ 4 ]    [  5 ]   [  6   ] [  -  ]|                            |
|  [ 1 ]    [  2 ]   [  3   ] [  +  ]|                            |
|  [ +/-]   [  0 ]   [  .   ] [  =  ]| [清除历史记录]              |
+------------------------------------+----------------------------+
```

## 4. 交互细节与状态说明 (Interaction Details)
* **[主题切换按钮]**：
  - **默认状态**：根据系统配色（媒体查询 `prefers-color-scheme`）自适应展示明亮（☀）或暗黑（🌙）主题。
  - **交互响应**：点击后平滑过渡切换主题（Transition time: 0.3s）。
* **[输入与操作按钮]**：
  - **触态与悬停**：PC端 Hover 时按钮背景色加深 10%；点击（Active/Click）时，按钮产生微弱按压内阴影并缩放（scale: 0.95），提供触觉确认感。手机端支持 100ms 的轻微震动反馈（若设备支持）。
  - **数字按钮与小数点**：
    - 输入数字时，主数值显示区字体根据字数自动微调（字数超 10 位时自动缩放字体大小，以防溢出）。
    - 小数点（.）校验：一次输入序列中只允许存在一个有效小数点。若当前输入数已含小数点，再次点击/输入小数点时，按钮应短暂闪烁红色（报错反馈）且不录入。
  - **运算符按钮**：
    - 选中某一运算符时，该按钮高亮显示，指示当前正处于的计算状态。
    - 若在输入数字前点击运算符，默认对 "0" 进行操作。
    - 若连续点击不同运算符，将后一个运算符覆盖前一个（例如依次输入 `+` 和 `-`，最终以 `-` 为准）。
* **[等号 (=)]**：
  - 点击后计算得出结果。若无第二操作数，则重复执行上一次的操作。
  - 成功计算后，结果写入主显示区，完整算式写入上方公式显示区，并异步将记录 `[算式 = 结果]` 追加到历史记录面板顶部。
* **[退格/回退按钮]**：
  - 单击清除当前输入数值的最后一位。若主数值显示区仅剩一位，则回退后显示 "0"。
  - 长按（> 0.8s）可快速清除当前输入（等同于 CE 按钮效果）。

## 5. 异常流与兜底设计 (Exception Flows)
* **除数为零 (Division by Zero)**：
  - **交互逻辑**：当进行类似 `x / 0` 操作并点击 `=` 时，主数值显示区不应崩塌，而应显著显示 "不能除以0"。
  - **状态限制**：此时除 "C"、"CE" 和 "退格" 按钮外，其他数字与运算按钮暂时置灰禁用，直到用户清除当前错误状态。
* **数值溢出 (Overflow)**：
  - **交互逻辑**：计算结果超出计算器所能呈现的最大范围（例如大于 1e15 或小于 -1e15）时，主数值显示区以科学计数法（如 `1.2345e+15`）显示。如果超出极限值（Infinity），显示 "数值溢出"。
* **空历史状态 (Empty History)**：
  - **交互逻辑**：当历史记录面板中无任何计算记录时，不展示空白列表，而是展示提示文字 "暂无计算历史"（低饱和度置灰文字，居中排版）。

## 6. HTML 原型实现约束
* **自适应框架**：原型必须使用 CSS Grid/Flexbox 响应式布局，保证在 320px 到 1920px 宽度的设备上均能居中且比例正确地展示。
* **状态切换模拟**：
  - 必须完整模拟明亮/暗黑主题的切换。
  - 小数点输入防重复、除零报错、溢出展示等 JS 逻辑必须在原型中直接实现，以便在评审中真实体验异常交互流。
  - 历史记录的追加与清除功能须利用内存（或本地 localStorage）进行简单模拟。
