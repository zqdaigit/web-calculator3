# 交互设计说明书: 计算器体验优化与Bug修复

## 1. 业务目标与用户画像
* **目标**：优化简易计算器的使用体验，修复当前版本在物理键盘映射、大屏显示器适配以及单数字 Enter 键触发计算三个层面的交互设计漏洞，使用户能高效、直观地在 PC 端和各主流分辨率下完成计算任务。
* **核心用户**：频繁使用 PC 物理键盘输入进行数值计算、对大屏无滚动条适配有高要求、关注计算流程规范度的普通及专业用户。

## 2. 页面跳转流程 (User Flow)
* **物理键盘输入与反馈流程**：
  - 用户聚焦页面 -> 输入物理按键 -> 识别映射关系 -> 虚拟按键触发 `active` 状态动画（100ms）-> 输入渲染至公式区/结果区。
  - 用户按下不支持的无效物理键 -> 页面不作响应，保持当前输入状态。
* **等号与回车（Enter）触发条件流程**：
  - 用户只输入单个数字 -> 点击 `=` 或物理 `Enter` -> 界面无变化，计算器处于等待状态，不产生计算动作，不计入历史记录。
  - 用户输入有效算式（如 `5 + 3`） -> 点击 `=` 或物理 `Enter` -> 输出计算结果 `8` -> 追加记录至历史记录面板顶部。
  - 用户仅输入操作数与运算符（如 `5 +`） -> 点击 `=` 或物理 `Enter` -> 依据默认右操作数规则（如补 `5`）进行计算 -> 输出结果 -> 追加记录至历史记录面板顶部。

## 3. 信息架构与页面布局 (Wireframe)

### 3.1 PC 端自适应无滚动条布局 (Tablet & PC: >= 768px, 针对 1920x1080 优化)
视口锁定高度为 `100vh`，禁用全局滚动条，采用两栏弹性布局，计算器主面板与历史记录面板自适应缩放。

## 4. 交互细节与状态说明 (Interaction Details)

* **[物理键盘按键映射与反馈]**：
  - **映射关系**：
    - 物理数字键 `0-9` 映射至虚拟数字键。
    - 物理小数点 `.` 映射至虚拟小数点。
    - 物理运算符 `+`, `-`, `*` (或 `x`), `/` 分别映射至虚拟键 `+`, `−`, `×`, `÷`。
    - 物理 `Enter` 键和等号 `=` 键映射至虚拟键 `=`.
    - 物理退格 `Backspace` 键映射至虚拟键 `退格`.
    - 物理清除 `Escape` 键或 `Delete` 键映射至虚拟键 `C`.
  - **按键视觉反馈**：
    - 当监听到合法的物理按键按下（`keydown`）时，界面上对应的虚拟按键立即被添加瞬时高亮效果（如背景色变暗/变亮 20%、加入轻微投影、`transform: scale(0.95)` 缩放效果）。
    - 按键弹起（`keyup`）或在 100ms 后，恢复按键的常规状态，为键盘输入用户提供与点击相同的物理操作反馈。
  - **操作指引**：
    - 界面底部显著且不遮挡操作的区域常驻一排物理键盘操作指南，字体为浅灰低对比度，提示有效按键范围。

* **[PC 端分辨率自适应规范]**：
  - **视口高宽锁定**：外层容器设置 `height: 100vh; overflow: hidden; width: 100vw;`。任何横向或纵向的滚动条均不允许出现。
  - **组件高占比缩放**：
    - 计算器外卡片 `.calculator-card` 在大分辨率下居中对齐，最大高度限制为 `calc(100vh - 120px)` 以留出页眉和页脚的间距。
    - 按键区域 `.keypad` 使用 CSS Grid，单元格高度使用弹性比例 `grid-template-rows: repeat(5, 1fr)` 自动撑满剩余卡片空间。
    - 历史记录面板 `.history-panel` 采用 flex 直向布局，历史记录列表 `.history-list` 采用 `flex-grow: 1; overflow-y: auto;`。当历史记录条目过多超出区域高度时，仅在 `.history-list` 区域内产生局部垂直滚动条，其余面板部分及全局视口不产生任何滚动条。

* **[单数字输入 Enter 键响应规范]**：
  - **判定规则**：
    - 当用户仅输入一个有效数值（包含正负号与小数点，如：`5`、`-12`、`3.14`），且输入缓冲区未录入任何二目运算符（`+`, `−`, `×`, `÷`）时。
    - 此时若用户点击界面 `=` 键或按下物理 `Enter` 键，计算器**不触发计算流程**，**公式显示区与主显示区界面完全无变化**，**且不追加任何计算记录至历史记录面板**。
  - **交互保护**：
    - 避免单纯输入数字的回车被污染录入历史记录，保持历史记录只记录有运算关系的式子。
    - `=` 键不置灰，不给用户造成“按键失效”的坏体验。当用户补全算式后（如再次输入 `+` 等），则立即恢复 `=` 和 `Enter` 键的标准求值流程。

## 5. 异常流与兜底设计 (Exception Flows)
* **无效按键防护**：用户输入无效字符时（例如字母、特殊符号），界面维持原样，输入框与公式区不累加非算术内容，不给出阻断性的强弹窗报错，仅保持输入拦截。
* **单数字 Enter 操作指引**：如用户在仅输入单数字时连续多次按 `Enter` 超过 3 次，可在主显示区下方浮现一个弱气泡提示：“请输入运算符（+、-、*、/）后再按 Enter 键计算”，并在 2 秒后自动消失，提升对新手的操作引导性。

## 6. HTML 原型实现约束
* 原型页面高度布局必须采用 `flex` 或 `grid` 动态分配高度，在大分辨率如 1920x1080 浏览器全屏时验证无滚动条。
* 键盘按下反馈的视觉 `active` 动效样式须在 CSS 中使用类选择器 `.active` 支持，并由 JS 拦截 `keydown` 和 `keyup` 事件控制类的添加与移除。
* JS 逻辑中需记录当前表达式的合法运算符数量，若无二目运算符则阻断 `=` / `Enter` 的历史记录写入。
